# Minecraft協調学習システム - システム概要

## 📋 システム全体像

Minecraft協調学習システムは、Minecraft Forge 1.20.1とScratch 3.0を連携させた教育支援システムです。インターネット環境の有無に関わらず、安定した協調学習体験を提供します。

### 🎯 主要目標
- **協調学習の促進** - Minecraftを通じた共同作業とコミュニケーション
- **段階的学習支援** - 初心者から上級者まで対応する教育カリキュラム
- **教師支援機能** - 教室管理と学習進捗の可視化
- **多様性への対応** - 8言語対応とアクセシビリティ配慮
- **安定性の確保** - オフライン対応と堅牢なエラーハンドリング

## 🏗️ システム構成

### Core Components (コアコンポーネント)

#### 1. **SystemController** 
- システム全体のライフサイクル管理
- コンポーネント間の統合制御
- パフォーマンス監視と最適化

#### 2. **ErrorManager**
- 包括的なエラーハンドリング
- エラー分類と重要度管理
- リアルタイム通知システム

#### 3. **CacheManager**
- 高速データアクセス
- メモリ使用量最適化
- パフォーマンス統計管理

### Feature Modules (機能モジュール)

#### 4. **CollaborationManager**
- プレイヤー間の招待・訪問システム
- リアルタイム協調作業支援
- 安全な帰還機能

#### 5. **TeacherManager**
- 教室モードとアクセス制御
- 学生活動のリアルタイム監視
- 権限管理と制限設定

#### 6. **ProgressTracker**
- 学習進捗の自動追跡
- 達成度とマイルストーン管理
- 個別化学習支援

#### 7. **LanguageManager**
- 8言語での完全ローカライズ
- 動的言語切替
- 地域設定対応

#### 8. **BlockPackManager**
- 教育目的別ブロック制限
- カスタムパック作成
- 段階的学習支援

#### 9. **OfflineModeManager**
- ネットワーク非依存学習
- データ同期とバックアップ
- 学習継続性の保証

#### 10. **AgentManager**
- AIアシスタント機能
- 学習ガイド提供
- インタラクティブサポート

## 🔧 技術仕様

### 開発環境
- **Minecraft Forge**: 1.20.1-47.2.0
- **Java**: 17
- **Gradle**: 8.1.1
- **Node.js**: 18+ (Scratch拡張機能)

### 通信プロトコル
- **WebSocket**: localhost:14711 (メイン通信)
- **HTTP**: REST API (設定・管理)
- **JSON**: データ交換フォーマット

### データ管理
- **ローカルファイル**: JSON形式での永続化
- **メモリキャッシュ**: 高速アクセス用
- **同期機能**: オンライン/オフライン自動切替

## 🎓 教育機能詳細

### 学習進捗管理
```
学習段階: 初心者 → 中級 → 上級 → エキスパート
追跡項目: 建築回数、協調作業、創造性指標、技術習得
評価方式: マイルストーン達成、ピアレビュー、自己評価
```

### ブロックパック体系
```
基本パック      : 最小限のブロック（10種類）
教育パック      : 色彩学習用ブロック（19種類）
初心者パック    : 学習開始用ブロック（10種類）
クリエイティブ  : 創造性重視ブロック（18種類）
建築パック      : 建築専門ブロック（20種類）
レッドストーン  : プログラミング学習（19種類）
自然パック      : 環境学習用ブロック（23種類）
上級者パック    : 高度機能ブロック（18種類）
```

### 多言語対応
```
対応言語: 日本語、英語、中国語（簡/繁）、韓国語、
         スペイン語、フランス語、ドイツ語
翻訳範囲: UI、エラーメッセージ、学習ガイド、
         達成度説明、ブロックパック説明
```

## 🚀 運用ガイド

### システム起動手順

1. **Minecraft Forgeサーバー起動**
   ```bash
   cd minecraft-mod
   ./gradlew runServer
   ```

2. **Scratch拡張機能ビルド**
   ```bash
   cd scratch-extension
   npm install
   npm run build
   ```

3. **システム初期化確認**
   - WebSocketサーバー接続確認 (port 14711)
   - 全コンポーネント初期化確認
   - エラーログ監視

### 教師向け運用手順

#### クラス開始時
1. **教師アカウント登録**
   - Scratchで「教師として登録」ブロック実行
   - 権限確認とクラスルーム設定

2. **学習環境設定**
   - 適切なブロックパック選択
   - 学生権限とアクセス制御設定
   - 言語設定確認

3. **学習活動開始**
   - 学生招待とグループ編成
   - リアルタイム活動監視開始
   - 学習目標とマイルストーン設定

#### クラス運用中
- **学生活動監視**: リアルタイム進捗確認
- **協調学習支援**: グループワーク促進
- **トラブル対応**: エラー検知と支援提供
- **評価記録**: 達成度とコメント入力

#### クラス終了時
- **進捗データ保存**: 学習成果の永続化
- **レポート生成**: 活動サマリー作成
- **次回準備**: 継続学習計画策定

### 学生向け利用手順

1. **システム接続**
   - Scratch拡張機能読み込み
   - Minecraft接続確認

2. **協調学習参加**
   - 招待受諾とワールド訪問
   - チームメンバーとの協力
   - 学習課題への取り組み

3. **進捗確認**
   - 個人達成度確認
   - 次の学習目標設定

## 🛠️ トラブルシューティング

### 一般的な問題と解決策

#### 接続問題
```
症状: WebSocket接続失敗
原因: ポート14711がブロックされている
解決: ファイアウォール設定確認、ポート開放
```

#### パフォーマンス問題
```
症状: 応答が遅い
原因: メモリ不足、キャッシュ無効
解決: キャッシュ最適化実行、メモリ増設検討
```

#### データ同期問題
```
症状: オフラインデータが同期されない
原因: ネットワーク不安定、同期エラー
解決: 手動同期実行、エラーログ確認
```

### エラーレベル別対応

#### Critical (重大)
- **即座に対応**: システム停止の可能性
- **エスカレーション**: 開発チーム連絡
- **一時対策**: オフラインモード切替

#### High (高)
- **1時間以内対応**: 機能制限の可能性
- **代替手段**: 関連機能での補完
- **監視強化**: 類似エラー発生注意

#### Medium (中)
- **4時間以内対応**: ユーザー体験への影響
- **ログ詳細確認**: 根本原因分析
- **予防策実施**: 再発防止措置

#### Low (低)
- **24時間以内対応**: 軽微な不具合
- **定期対応**: 次回メンテナンス時修正
