# 🔥 Firebase Hosting セットアップガイド

## 🎯 目標
Firebase Hostingで専用Scratchページを公開して、以下のようなURLでアクセス可能にする：
```
https://minecraft-scratch-12345.web.app/
```

---

## 📋 セットアップ手順（10分で完了）

### ステップ1️⃣: プロジェクト作成

**現在のページ**: https://console.firebase.google.com/

1. **「プロジェクトを作成」** をクリック

2. **プロジェクト名を入力**
   ```
   minecraft-scratch-system
   ```
   または好きな名前（英数字とハイフンのみ）

3. **Google アナリティクスの設定**
   - 「このプロジェクトでGoogle アナリティクスを有効にする」
   - **チェックを外してOK**（不要な場合）
   - または有効にして続行

4. **「プロジェクトを作成」** をクリック
   - 作成完了まで1-2分待機

### ステップ2️⃣: Hosting機能を有効化

プロジェクトが作成されたら：

1. **左サイドバーで「Hosting」** をクリック
   ```
   🏠 Hosting
   ```

2. **「始める」** をクリック

3. **Firebase CLI のインストール指示が表示**
   - この画面は**一旦スキップ**します
   - **「次へ」** をクリック

4. **プロジェクト初期化の画面**
   - **「次へ」** をクリック

5. **「コンソールに進む」** をクリック

### ステップ3️⃣: 手動アップロード（最も簡単な方法）

1. **Hostingページで「新しいサイト」** をクリック

2. **カスタムドメインの設定**
   - **スキップしてOK**
   - 後で設定可能

3. **「手動デプロイ」** を選択

4. **ファイルのアップロード**
   - **`D:\minecraft_collaboration_project\docs\`** フォルダ内の全ファイルを選択
   - ドラッグ&ドロップまたは「ファイルを選択」

### ステップ4️⃣: デプロイ完了

1. **アップロード完了を待機**（1-2分）

2. **デプロイURL取得**
   ```
   https://minecraft-scratch-system-12345.web.app/
   ```
   ※実際のURLは自動生成されます

3. **「サイトにアクセス」** をクリックして確認

## ⚡ 簡単版手順（要点のみ）

1. **Firebase Console** → **「プロジェクトを作成」**
2. **プロジェクト名**: `minecraft-scratch-system`
3. **左サイドバー** → **「Hosting」** → **「始める」**
4. **「手動デプロイ」** → **`docs/` フォルダの中身をアップロード**
5. **完了！** URLが自動生成されます

---

## 🎮 アップロードするファイル

以下のファイルを **`docs/` フォルダから** アップロード：

```
✅ アップロード対象ファイル：
├── index.html          # メインのScratchページ
├── (その他のファイル)   # docs フォルダ内の全て
```

**重要**: `docs/` フォルダ内の **`index.html`** が最重要ファイルです。

## ❓ トラブルシューティング

### 問題1: 「プロジェクトを作成」が見つからない
**解決**: 
1. Googleアカウントでログインしているか確認
2. ページを再読み込み
3. https://console.firebase.google.com/ に直接アクセス

### 問題2: アップロードに失敗する
**解決**: 
1. ファイルサイズが大きすぎないか確認（1ファイル10MB以下）
2. ブラウザを変更（Chrome推奨）
3. 1つずつファイルをアップロード

### 問題3: サイトが表示されない
**解決**: 
1. `index.html` がルートにあるか確認
2. 5-10分待ってから再度アクセス
3. ブラウザキャッシュクリア
